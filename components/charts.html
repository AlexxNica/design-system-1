Chart BasicsIn Progress

When considering using a data visualization in your project, always value substance over flash. The most important thing is making sure it is easy to interpret the data and glean insights. Charts can also present accessiblity chanlenges designers must address. Remember to keep in mind color blind users, keyboard accessibility, and users with screen readers.

Best practices include:

  • Labeling the chart and axises. Don't make the user guess what data points mean. The only case an axis label isn't needed is when the x-axis shows time.
  • Using a legand when differences in color and pattern convey information in charts.
  • Verifying chart colors can be differentiated by colorblind users. When possible, use patterns in addition to color to differentiate data.
  • Displaying a threshold or average line with the information is available. This give the user quick context as to whether they are doing better or worse than expected.
  • Verifying the information in the charts is also available to someone using a screen reader.
  • Never fully relying on hover tooltips to convey information.

Use: comparison between items or comparison thoughout time and identify outliers

Bar charts can help the user quickly identify high and low values in data sets. You can compare items against each other, such as in the first example. In the second example, the same item is compared overtime. A line chart could also be used in this scenario and works better if you are showing more than 12 data points. Bar charts work best if large differences between bars are expected. Subtle fluctuations will be harder to identify in a bar chart than in a line chart.

Bar in charts with a single data set should be the same color, and there is no need to show a legend.

Multiple Data Sets

Up to four data sets can be compared overtime using a bar chart. Each data set should be a unique color, and a legand must be displayed. Clicking on a data set label in the legend toggles between showing and hiding that information in the chart.

In order to show multiple data sets on the same bar chart, the same y-axis must be represent the same variable. If the y-axis variables are different, show the information in separate charts.Creating multiple y-axises can make charts misleading and cause confusion.

Use: show relationship or show three variables

A bubble chart displays three variables of data at a time. It can show the correlation between the x- and y-axis variables. A third variable determines the size of the bubble. Be conscientious of the number of bubbles and the size of the chart. Too many bubbles on too small of a chart can create a useless visualization.

A legend doesn't necessarily need to be shown with this chart. However, it can be used to filter what bubbles are displayed and therefore helpful in some circumstances. Make sure to include a custom scale indicating what bubble diameter indicates.

Line ChartIn Progress

Use: trends over time

Much of the guidence around bar charts also applies to line charts. Line charts with a single line do not need a legend. Up to six lines can be displayed on a single chart with a legend. Lines must be throroughly differentiated with color or pattern. The same y-axis variable must apply to all lines on the chart.

Polar Area ChartIn Progress

Use: identify outliers and display cyclical data

Polar area charts have similarities to both bar and pie charts. Unlike pie charts, they do not show an item's share of the total. Instead each wedge has an equal angle. Instead, the wedges extend from the center to show deviations in the data the way bars extend from the x-axis in the above bar chart examples. Polar area charts are best at showing cyclical data.

Since polar area charts are less common compared to bar and pie charts, gleaning insights from them can increase cognitive load. Depending on your data, consider showing a bar chart or pie chart instead.
